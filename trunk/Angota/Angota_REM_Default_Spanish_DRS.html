    3.1 Entrada pedido
    SOP-0001 comenzarPedido
    Tipo del resultado El valor del identificador del objeto de la clase Pedido que se ha creado
    Versión 1.0 ( 17/05/2010 )
    Autores
    • Rafael Vergara Moreno
    Dependencias
  • [OBJ-0001] Control de compras
  • [OBJ-0002] Control de ventas
    • Descripción Esta operación describe el comienzo del registro de un nuevo pedido en el sistema
    Parámetros DNICliente : String -- el dni correspondiente al cliente.
     DNIComercial : String -- el dni correspondiente al comercial que ha realizado el pedido.
     DNIAdministrador : String -- el dni correspondiente al administrador.
    numeroPedido : Integer -- el número que se le asigna al pedido.
    Expresiones de precondición Existencia de objeto de la clase Cliente en el sistema: Existe un objeto de la clase Cliente en el sistema tal que su identificador es DNICliente
     Existencia de objeto de Proveedor en el sistema: Existe un objeto de la clase Proveedor en el sistema tal que su identificador es proveedorID
    Expresiones de precondición (OCL) Existencia de objeto de la clase Cliente en el sistema: Pedido.allInstances() -> one(c|c.DNI = DNICliente)
    Existencia de objeto de Proveedor en el sistema: Pedido.allInstances() -> one(p|p.DNI = DNIProveedor)
    Expresiones de postcondición Creación de objeto de la clase Pedido: Se ha creado un objeto de la clase Pedido a la que se le han pasado como parámetros los datos del pedido, tal que su identificador es pedidoID
    Expresiones de postcondición (OCL) Creación de objeto de la clase Pedido: let ped : Pedido = Pedido.allInstances() -> any (numero = numPed), numPed: Integer = Pedido.allInstances() -> size()+1 in ped.oclIsNew() = true and ped.fecha = Date.Today() and return = ped.numero
    Comentarios Ninguno
